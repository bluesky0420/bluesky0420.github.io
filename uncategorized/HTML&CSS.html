HTML & CSS概述

1.web

2.W3C

3.xml

​ 可扩展的标记语言：extension markup language,用于定义文档结构。

4.什么是HTML

​ HTML是W3C定义的语言标准：用来描述页面结构的语言。

​ 结构：有什么东西，这个东西表示什么含义。

5.什么是CSS

​ CSS是W3C定义的语言标准：用来展示页面效果的语言。

​ CSS决定了页面长什么样子。

6.执行HTML CSS

​ HTML、CSS → 浏览器内核执行 → 页面

​ 浏览器：shell(外壳) + core(内核):js执行引擎、渲染引擎、其他模块

​ IE：Trident

语义化

1.什么是语义化

​ 每一个HTML元素都具体有具体含义

​ 所有HTML元素与展示在页面的效果无关,页面效果由CSS决定

​ 当我们只有HTML元素而没有设置CSS属性但页面仍有效果是因为浏览器有默认的CSS样式

2.为什么需要语义化

​ 为了搜索引擎优化(SEO)

​ 每隔一段时间搜索引擎会从整个网页获取网页源码

​ 为了让浏览器理解网页

​ 阅读模式、语音模式

元素的包含关系

1.之前的说法

​ 之前的说法是，块级元素可以包含行级元素，行级元素不能包含块级元素，a元素除外

​ 现在因为语义化的提出，上面的说法失效，因为html元素与它展现页面的效果无关

2.总结

​ 1.容器元素中可以包含任何元素

​ 2.a元素中可以包含几乎所有元素

​ 3.某些元素有固定的子元素如：ul > li…..

​ 4.标题元素和段落元素不能互相嵌套，且不能包含容器元素

盒模型

1.概念和相关的元素

2.盒子类型

​ 行盒，display等于inline的元素

​ 块盒，display等于block的元素

​ display默认值是inline

​ 浏览器默认的样式表设置的块盒：容器元素、h1-h6，p,div

​ 常见的行盒：span、a、img、video、audio

3.盒子的组成部分

​ 无论行盒还是块和都由以下几个部分组成

​ 1.content：width，height，设置盒子的宽高，内容部分内容盒content-box

​ 2.padding：填充，盒子边框到盒子内容的距离，left，right，top，buttom上右下左四个方向一样只写一个数，上下相同，左 右相同只写两个 填充区+内容区 = 填充盒padding-box

​ 3.border：边框，有三个属性，style，color，width边框 = 边框样式 + 边框大小 + 边框 颜色 边框区+填充区+内容区=边框盒 border-box

​ 4.margin：外边框，用来设置和其他盒子的距离

行盒盒模型

1.概念

​ 常见行盒：包含具体内容的元素

​ span、strong、em、i、s、del、img、video、audio

2.显著特点】

​ 盒子沿着内容延申

​ 行盒不能设置宽高

​ 调整行盒的高度，应该使用字体大小，行高，字体类型，间接调整。

​ 内边距，边框，外边距水平方向上有效，垂直方向上仅会影响背景，不会占据实际空间。

3.行块盒

​ display：inline-block；

​ 不会独占一行

​ 块盒盒模型中的所有尺寸都有效。

​ 空白折叠，发生在行盒内部 或 行盒之间（包括行块盒）

​ 可替换元素：img、video、audio（补充：object-fit：属性；可以设置图片的适应模式）

​ 绝大部分的可替换元素均为行盒，类似于行块盒可以设置宽高

​ 非可替换元素：页面上显示的效果，取决于元素的属性

常规流

1.基本概念

​ 盒模型：固定单个盒子的规则

​ 视觉格式化模型：页面上多个盒子排序规则。

​ 视觉格式化模型，大体将盒子排序分为三类：1.常规流 2.浮动 3.定位

​ 包含块（containing block）：每个盒子都有它的包含块，包含块决定了盒子的排列区域。

​ 绝大多数情况下：盒子的包含块，为其父元素的内容盒。

2.常规流布局

​ 概念

​ 常规流、文档流、常规文档流、普通文档流

​ 所有元素，在默认情况下，都属于常规流布局

​ 总体规则：块盒独占一行，行盒依次排序

3.块盒在常规流中的规则

​ 每个块盒的总宽度，必须刚好等于父元素的内容盒，宽度的默认值是auto（将剩余空间吸收）margin的取值是auto默认值是 0，且宽度的吸收能力强于margin，若宽度、内边距、边框、外边距计算后，仍有空间，该空间被margin-right吸收

​ 在常规流中，块盒在包含快中居中，可以定宽，然后设置左右margin为auto。

​ 每个块盒垂直方向上的auto值

​ height：auto；适应内容高度

​ margin：auto；表示0

​ 百分比的取值

​ 1）padding、宽高、margin可以取值为百分比

​ 2）包含块的高度是取决于子元素的高度，设置百分比无效。

​ 3）包含块的高度不取决于子元素的高度，百分比相对于父元素的高度。

​ 上下外边距合并

​ 1）两个常规流块盒，上下外边距相邻，会进行合并。

​ 2）相邻外边距合并取最大值。

浮动

1.应用场景

​ 文字环绕

​ 横向排列

2.浮动的基本特点

​ 设置float属性值为：

​ -left：左浮动，元素靠上靠左

​ -right：右浮动，元素靠上靠右

​ 默认值为：none

​ 当一个元素浮动后，元素必定为块盒（更改属性display：block；）

​ 浮动元素的包含块，盒常规流一样，为父元素的内容盒

3.盒子尺寸

​ 1）宽度为auto时，适应内容宽度

​ 2）高度为auto时，适应内容高度

​ 3）margin为auto时在浮动里面四边都为零

​ 4）边框、内边距、百分比设置和常规流一样

4.盒子排列

​ 1）左浮动盒子考上靠左排列

​ 2）右浮动盒子考上靠右排列

​ 3）浮动盒子在包含块中排列时会避开常规流块盒

​ 4）常规流块盒在排列时，会无视浮动盒子

​ 5）行盒在排列时会避开浮动盒子

​ 6）外边距不会发生浮动

​ 补充知识：如果文字没有在行盒中，浏览器会生成一个盒子将文字包裹起来，表示文字永远在行盒中，这个盒子叫做匿名行盒

5.高度坍塌

​ 高度坍塌根源：常规流盒子的自动高度（未设置高度情况下），在计算时，会忽略浮动盒子。

​ 清除浮动

​ 涉及css属性：clear 默认值是none，属性有left，right，both

​ -left：清除左浮动，必须出现在所有左浮动盒子的下方

​ -right：清除右浮动，必须出现在所有右浮动盒子的下方

​ -both：清除左右浮动，必须出现在所有浮动盒子的下方

​ 解决方法：

​ 1）在所有浮动盒子的父元素上添加一个内样式clearfix设置clearfix::after{content：””;dispaly：block；clear：both；}

​ 2）在所有浮动元素后面添加一个元素并设置高度样式（不推荐）

​ position属性可以手动控制元素在包含块中的精准位置

​ 默认值是：static，静态定位（不定位）

​ 属性值：

​ -relative：相对定位，相对于它自身

​ -absolute：绝对定位，相对于它祖先元素中第一个定位元素的填充盒。若没有则相对于整个网页（初始化包含块）

​ 其他情况下和绝对定位一样

​ 不同：包含块为视口（浏览器可视窗口）

5.定位下的居中

​ 没有外边距合并

​ 常规流块盒的布局

2.块级格式化中常规流块盒的布局规则

​ 1）常规流块盒在水平方向上，必须撑满包含块

​ 2）常规流块盒在垂直方向上依次摆放

​ 3）常规流块盒若外边距无缝相连，会进行合并

​ 2）设置了浮动、绝对定位、固定定位的元素

​ 3）overflow不等于visible的块盒

4.BFC元素的规则

​ 独立的：不同的BFC区域在渲染的时候互不影响，创建了BFC的元素，隔绝了内部和外部的联系，内部渲染不会影响到外部。

​ 创建了BFC的元素，会计算浮动元素的自动高度。

​ 创建了BFC的元素，他的边框不会和浮动元素重叠。

​ 创建了BFC的元素，不会和其他子元素进行外边距合并。

堆叠上下文

1.概念

​ 堆叠上下文（stack context）：是一块区域，这块区域由某个元素创建，它规定了该区域中的内容在z轴上的先后顺序。

2.创建堆叠上下文的元素

​ 1）html元素（根元素）

​ 2）设置了z-index数值（非auto）的定位元素

3.同一个堆叠上下文中的元素在z轴上的排列顺序

​ 从后到前的顺序：

​ 1）创建堆叠上下文的元素的背景和边框

​ 2）堆叠级别为负值的堆叠上下文

​ 3）常规流非定位的块盒

​ 4）非定位的浮动盒子

​ 5）常规流非定位的行盒

​ 6）任何z-index是auto的定位子元素，以及z-index为0的堆叠上下文

​ 7）堆叠级别为正值的堆叠上下文

4.特点

​ 每个堆叠上下文，独立于其他堆叠上下文，他们之间不能互相穿插

​ 在同一个堆叠上下文元素中z-index值越大越靠前，值越小越靠后，值相等后面的覆盖前面的。

​ 不同的堆叠上下文，要把它当作一个整体，同级别中书写靠前的会覆盖书写靠后的

数据连接data url

1.如何书写

​ 数据链接：将目标文件的数据直接书写到路径中

​ 语法：data：MIME/base64，数据

2.意义

​ 减少了浏览器的请求

​ 优点：

​ 1）减少了请求中浪费的时间

​ 2）有利于数据的动态生成

​ 缺点：

​ 1）增加了资源的体积，导致传输内容增加，从而增加了单个资源的传输

​ 2）不利于浏览器缓存，浏览器通常会缓存图片文件，css文件，js文件。

​ 3）会增加原资源的体积到原来的四分之三

​ 应用场景：

​ 1）当请求单个图片前提小，并且该图片因为各种原因，不适合制作雪碧图，可以使用数据链接。

​ 2）图片由其他代码动态生成，并且图片较小。

3.base64

​ 表示的是一种编码方式

​ 通常用于将一些二进制数据用一个可书写的字符串表示

​ 图片是一个二进制数据

​ tip：css的数据类型是text，通常在数据链接中的图片资源的MIME后加一个分号后加一个base64

浏览器的兼容性

1.问题产生原因

​ 1）市场竞争导致不同浏览器为了市场竞争制定了不同的版本，但是官方定义的速度较慢导致了浏览器出现了兼容性问题

​ 2）标准版本更新变化导致浏览器出现了兼容性问题

​ 3）浏览器的不同版本也会出现一些兼容性问题，老版本的浏览器不支持新版本中的元素

2.厂商前缀

​ 原因：

​ 1）市场竞争，标准文档还没有发布

​ 2）标准仍在讨论中，浏览器厂商希望先支持

​ 常见的厂商前缀

​ IE：-ms-

​ Chrome，safari：-webkit-

​ firefox：-moz-

​ opera：-o-

​ 为了兼容低版本浏览器可以将厂商前缀加上

​ 例子：

​ box-sizzing，谷歌旧版本浏览器的书写形式是-webkit-box-sizzing：厂商前缀说明这个元素在当时官方还没有发布

​ 谷歌浏览器的滚动条样式

​ 实际开发中往往是div+css+js来实现滚动条，例如一个div里面放置一个div用css设置来设置他的样式，js来控制滑动

​ 多背景中选择一个作为背景

3.css hack（补丁）

​ 根据不同的浏览器（主要针对于IE浏览器）设置不同的样式和元素

​ 1）样式

​ IE中，css的特殊前缀

​ “*” 属性，兼容IE5,IE6,IE7

​ “_”属性，兼容性IE5,IE6

​ 属性值后面加\9，兼容IE9,IE10

​ 属性值后面加\0，兼容IE-IE10

​ 属性之后面加\9\0，兼容IE9-IE10

​ IE6的外边框bug，浮动元素的左边距会翻倍

​ 2）判断条件

​ IE浏览器能识别注释内的一部分元素

4.渐进增强优雅降级

​ 渐进增强：先适应大部分浏览器，然后针对新版本浏览器，加入新的样式，书写代码是会先尽量避免有兼容性问题的代码，完 成后，在逐步加入新标准中的代码

​ 优雅降级：先制作完成功能，然后针对低版本浏览器进行特殊处理，在解决兼容性问题，书写代码时，先不用特别在意兼容 性，完成整个功能，在针对低版本浏览器处理样式。
