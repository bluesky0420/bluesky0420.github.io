JavaScript介绍、入门

1.web 发展史

​ Mosaic，是互联网史上第一个普遍使用和能够显示图片的网页浏览器。1993年问世。

​ Netspace Navigtor 浏览器

​ IE，Mozilla Firefox都是基于mosaic浏览器源码编写的。

​ JavaScript是Netspace Navgitor 浏览器的一部分首次出现在1996年。它最初的设计目的是改善网页的用户体验

​ 作者：Brendan Eich

2.浏览器组成

​ shell

​ core(渲染引擎，js引擎，其他模块)

3.Js的逼格

​ 解释型语言

​ 单线程，JavaScript引擎是单线程的

4.Js的三大部分

​ ECMAscript

​ DOM

​ BOM

5.知识补充

​ 解释型语言和编译型语言

​ 解释型语言：读一行翻译一行，翻译一行执行一行。优点：跨平台。不足：较慢

​ 编译型语言：通篇翻译，系统执行翻译文件。 优点：快。 不足：移植性不好

​ 单线程：一个执行体同一时间只能干一件事

​ 多线程：一个执行体同一时间可以干多件事

​ 同步：同一时间干一件事

​ 异步：同一时间干多件事

​ DOM：document object model 文本对象模型

​ BOM：broswer object model 浏览器对象模型

Js运算符

1算数运算符

​ “+”、”-“、”*”、”/“、”%”、”++”、”–”

​ %：取模(取余)

​ ++：数值前先加在算；数值后先算后加

​ –：和++同理

2赋值运算符

​ “=”、”+=”、”-+”、”*=”、”/=”、”%=”

​ =优先级最弱，()优先级最强

3比较运算符

​ “>=”、”<=”、”<”、”>”、”==”、”!=”

​ 比较的是Ascll，如果是字符串比较位数相同比较他们的Ascll，尾部不同，先比较第一位，看结果在比较后面的

​ 比较运算符最后得到的结果是布尔类型的

4逻辑运算符

​ “&&”、”||”、”!”

​ 逻辑运算符在比较的时候会把他们所得到的值转换成布尔形式的值

​ &&：第一个值为真，就看第二个值，并输出第二表达式得出的值，第一个为假时直接返回第一个表达式，通常用来中断后 续的运算过程

​ ||：第一个为真就返回第一个，第一个为假就返回第二个，通常用来解决浏览器的兼容性问题

​ ！：就是否的意思真为假，假为真

typeof、类型转换

1.typeof操作符

​ typeof可以返回Number，String，undefined，Boolean，object，function

​ array，object，null类型都会返回object

2.类型转换

​ 显示类型转换

​ Number（mix）：转换成数字类型

​ null转换值为0，布尔类型的true为1，false为0，undefined转换为NAN，只能转换带有数字的类型，非数字类型 转换结果为NAN

​ parseInt（String，radix）：转换成整形

​ 只能转换带有数字的类型，并只截取数字部分进而转换成整形。其他没有带有数字的类型转换得到的结果是NAN如 null，undefined转换结果为NAN

​ parseInt可以转换进制，可以将变量以多少进制转换成十进制。可实现的进制范围是2-32

​ parseFloat（String）：转换成浮点型

​ 只能转换带有数字的类型，并只截取数字部分转换为浮点型，且浮点型遵循四舍五入原则， 其他非带有数字类型 转换结果为NAN

​ toString（radix）：转换成字符型

​ 除了undefined和null以外可以将任何类型转换成字符串类型，undefined和null的转换会报错

​ toString可以转换成进制数，但是是将转换值转换成多少进制进行输出且输出形式为字符串形式

​ Boolean（）：转换成布尔类型

​ 除了undefined，null，””，false，NAN，0以外的值转换成布尔型都为真

​ String（mix）：转换成字符型

​ 理论上可以将任何原始值类型转换成字符串类型

​ 隐式类型转换

​ isNaN() → number

​ ++/– +/- * / % → number

​ && || ! < > <= >= == != → Boolean

​ isNaN()的内部原理为：

​ “+ “前后如果有字符串类型，隐式转换为字符串类型

​ 赋值运算从右到左，算数运算从左到右，()优先级最高

​ 隐式类型转换中undefined == null 的结果为true

​ 隐式类型转换中的NaN == NaN的结果为false

​ === 和 ! ==不会发生类型转换，且只有类型相同值相同的时候才会为真

​ typeof(typeof(a)) 的结果为String

作用域、作用域链

1.作用域

​ [[scope]]：每个JavaScript都是一个对象，对象中的有些属性我们能够访问有些属性我们不能访问，这些属性仅供JavaScript 引擎存取，[[scope]]就是其中一个，[[scope]]就是我们所说的作用域，其中存储了执行期上下文。

2.作用域链

​ [[scope]]中所存储的执行期上下文对象的集合，这个集合呈链式连接，我们吧这种链式连接叫做作用域链

3.执行期上下文（运行期上下文）

​ 当函数在执行的时候，会创建一个称为执行期上下文的内部对象，一个执行期上下文定义了一个函数执行时的环境，函数每 次执行时的执行期上下文都是独一无二的，多以多次调用一个函数会导致创建多个执行期上下文，当函数执行完毕时，执行 期上下文会自动销毁。(垃圾回收机制)

4.知识补充

​ 查找变量的方式是从作用域链的顶部依次往下查找即从AO向GO查找

​ 上级函数执行时，下级函数才会被定义

立即执行函数

​ 该函数没有声明，执行完后就被释放，适合做初始化工作

包装类

1.知识铺垫

​ 1）JavaScript数据类型(值类型)

​ 原始值类型：Number，String，Boolean，undefined，null

​ 存储在栈(stack)内存中的简单数据段，他们的值直接存储在变量访问的位置。

​ 注意：在众多语言中字符串都被看做是引用类型，而非原始类型，因为字符串长度可变，但是ECMAScript打破了这 一传统。

​ 引用值类型：Array，Object，function，Date，RegExp

​ 存储在堆(heap)内存中的对象，存储在变量空间中的值是一个指针(引用)指向存储对象的内存处。

​ 2）栈和堆

​ 栈：原始值存储在栈中，为变量赋值时，ECMAScript解析程序必须判断该值是什么类型，原始值类型占据的空间是固定 的，所以将他们存到较小的内存区域中，便于迅速查寻变量的值。

​ 堆：引用值存储在堆中，因为引用类型的大小会发生变化，所以不能存入栈中，否则会降低查寻的速度。所以栈存储地 址，堆存储数据。

​ 3）原始值和引用值的不同

​ 赋值方式，值是否可变

​ 比较方式不同

2.包装类产生原因

​ Js中只有对象才有属性和方法，原始值没有属性和方法，平时求字符串长度的属性length没有问题是因为Js给我们提供了三 种特殊的引用类型（Number String Boolean）每当我们给原始值添加属性的时候，后台会自动给我们进行包装类

原型，原型链

1.知识铺垫

​ 1）函数对象

​ 所有引用类型(函数、数组、对象)都拥有proto属性(隐式原型)

​ 所有函数拥有prototype属性(显示原型)(仅限函数)

​ 原型对象：拥有prototype属性的对象，在定义的时候就被创建

​ 2）构造函数

​ 是一种特殊的方法，主要用来创建对象时初始化对象。每个构造函数都有prototype(原型)属性

​ 必须使用关键字new和大驼峰式命名的才是函数

​ 3）原型模式

​ 每个函数都有prototype(原型)属性，这个属性是一个指针，指向一个对象，这个原型对象是用来给实例共享属性和方法

2.原型

​ 1）定义

​ 原型是function的一个属性，它定义了构造函数制造出来的对象的公共祖先。

​ 通过该构造函数产生的对象，可以继承该原型的属性和方法。原型也是对象。
