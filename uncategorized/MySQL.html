MySQL

单元一：基础部分

一、为什么学习数据库

​ 1）持久化数据到本地

​ 2）可以实现结构化查询，方便管理

二、数据库的相关概念

​ 1）DB：数据库（database）：存储数据的“仓库”。它保存了一系列有组织的数据。

​ 2）DBMS：数据库管理系统（Database Mangement Ststem）。数据库是通过DNMS创建和操作的容器

​ 3）SQL：结构化查询语言（Structure Query Language）：专门用来于数据库通信的语言

​ SQL特点：

​ 1.不是某个特定的数据库供应商专用语言，几乎所有DBMS都支持SQL

​ 2.简单易学

​ 3.灵活运用语言元素，可以进行非常复杂和高级的数据库操作。

​ 4）常见的数据库管理系统：MySQL，Oracle，DB2，SQL server

三、数据库存储数据特点

​ 1）将数据放到表里面，再把表放到库里面。

​ 2）一个数据库中可以有多个表，每个表都有一个名字，用来标识自己。表名具有唯一性。

​ 3）表具有一些特性，这些特性定义了数据在表中如存储，类似Java中“类”的设计。

​ 4）表由列组成，我们也称为字段。所有表都是由一个或多个列组成，每一列类似于Java中的”属性“。

​ 5）表中的数据是按行存储的，每一行类似于Java中的”对象“。

四、初识MySQL

MySQL产品介绍

​ 1）MySQL数据库隶属于MySQLAB公司，总部位于瑞典，后被Oracle收购

​ 2）优点：

​ 成本低：开放源代码，一般可以免费使用

​ 性能高：执行快

​ 简单：容易安装使用

MySQL安装

​ 1）DBMS分类：

​ 基于共享文件系统的SBMS（Access）

​ 基于客户机—服务器的DBMS（MySQL，SQLserver，Oracle）

​ 2）将所有软件终止进程

MySQL服务的启动和停止

​ 停止MySQL服务：net stop 对应的mysql的版本

​ 启动MySQL服务：net start 对应的mysql的版本

MySQL服务的登录和退出

​ 1）MySQL自带的客户端，输入root用户密码，exit/Ctrl+C退出（不建议使用，只能用于root用户，不够灵活）

​ 2）利用dos窗口输入mysql -h localhost -P 3306 -u root -p （h(host)：代表主机名,P(端口号)u(用户名)p(密码)）

​ 【注意】如果在dos窗口输入命令显示无效说明环境变量配置有误，将MySQL中的bin文件的目录添加到path的最前面

​ 2）打开指定的库：use 库名；

​ 3）查看当前库的所有表：show tables；

​ 4）查看其他库的所有表：show tables from 库名；

​ 5）创建表：create table 表名（

​ 列名 类型，

​ 列名 类型，

​ …………….

​ ）

​ 6）查看表结构：desc 表名；

​ 7）查看表的内容：select * from 表名；

​ 8）查看服务器版本：mysql：select version()；dos：mysql –version

2.语法规范

​ 1）不区分大小写，但建议关键字大写，表名，列名小写。

​ 2）每条命令最好用分号结尾

​ 3）每条命令根据需要，可以进行缩进或换行。

​ 4）注释：单行注释：#注释文字 – 注释文字 多行注释/* 注释文字 */
